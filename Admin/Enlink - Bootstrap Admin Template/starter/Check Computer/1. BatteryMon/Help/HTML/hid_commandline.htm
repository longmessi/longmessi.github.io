Command line options

    

There are two command line options. If no command line option is specified, BatteryMon starts normally.

/r

BatteryMon will automatically start taking samples when it is run, removing the need to click ‘start’.

/m

BatteryMon will shrink to the mini monitoring mode on startup. This must be used in conjunction with /r as the mini mode cannot be used unless BatteryMon is currently taking samples.

/d:xx:yy:zz

Upon launch BatteryMon will check to see if the battery level is above xx%. If it isn't then it will quit and log what happened. If it is then it will continue running until the battery is below yy% then quit with a message in the log. This also starts monitoring on startup the same as /m and forces the use of automatic logging. Once the battery has discharged to the required level and quit the log file will either report a success or failure based on zz. zz is the time in minutes that is the minimum time for the battery to discharge the amount set, if it took less time then this the test is reported to be a failure.

/L

Use the executable directory for the config file instead of the windows user data folders.

/c

Checks the battery level and then exits. The exit code is set to the current battery charge % for external application to read.

Example – Execute BatteryMon with automatic start and mini mode

Using “Run” selected from the Windows start menu,

"C:\BatteryMon\batterymon.exe" /r /m
